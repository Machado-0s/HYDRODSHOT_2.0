

# HYDRODSHOT 2.0 | ГИДРОНАВТИКА МГТУ ИМ. БАУМАНА


## 🌊 Обзор проекта
Данный репозиторий содержит критически важное встроенное ПО (firmware) для АНПА (Автономного Необитаемого Подводного Аппарата) команды **«ГИДРОНАВТИКА»**. Система разработана для управления сложной конфигурацией из 10 движителей, обеспечивая высокоскоростное цифровое управление и обратную связь в реальном времени с управляющим компьютером Raspberry Pi.

Прошивка базируется на модульной объектно-ориентированной архитектуре C++, что гарантирует масштабируемость и надежность при выполнении сложных подводных миссий.

## 🏗 Структура репозитория

Проект организован таким образом, чтобы четко разделить низкоуровневые драйверы оборудования, логику управления и протоколы связи.

```text
├── Core
│   ├── Inc/             # Заголовочные файлы приложения и определения DShot
│   ├── Src/
│   │   ├── main.cpp     # Точка входа и основной конечный автомат приложения
│   │   ├── myCode/      # Уровень протокола: Структура системной шины и интеграция hydrolib
│   │   ├── dshot.c      # Логика управления: PID-регуляторы, масштабирование RPM и маппинг моторов
│   │   ├── dshot_A.c    # Драйвер оборудования: Параллельный DMA bit-bang и движок декодирования GCR
├── Drivers/             # Библиотеки STM32 HAL и CMSIS
├── cmake/               # Конфигурации сборки CMake и тулчейна
└── SAUVC_T1_T8.ioc      # Файл конфигурации STM32CubeMX
```

## ⚡ Техническая особенность: Параллельный Bidirectional DShot

Ключевой особенностью данной прошивки является **программно-реализованный стек Bidirectional DShot**. В то время как стандартные реализации ограничены количеством каналов аппаратных таймеров, данный проект управляет **10 моторами одновременно**, используя высокопроизводительную архитектуру DMA.

### 1. Мультипортовый параллельный Bit-Banging
Для обеспечения работы 10 моторов мы используем 5 одновременных потоков DMA, инициируемых таймерами **TIM1** и **TIM8**. Потоки передают заранее подготовленные буферы битов напрямую в регистры GPIO `BSRR`. Это позволяет обновлять состояние пинов на пяти разных портах в один и тот же тактовый цикл, обеспечивая наносекундную синхронизацию.

**Маппинг потоков DMA (из `dshot_A.c`):**
* **Port A (PA5, PA6):** DMA2 Stream 5 (TIM1)
* **Port B (PB0, PB10):** DMA2 Stream 6 (TIM1)
* **Port C (PC6, PC8):** DMA2 Stream 4 (TIM1)
* **Port D (PD12, PD14):** DMA2 Stream 3 (TIM1)
* **Port E (PE9, PE13):** DMA2 Stream 1 (TIM8)

### 2. Декодирование GCR и замкнутый PID-контур
Bidirectional DShot требует от STM32 «слушать» ответ ESC, который закодирован с помощью **GCR (Group Code Recording)**.
* **Декодер:** `dshot_A.c` выполняет оверсемплинг входящего сигнала и программно декодирует 5-битные символы GCR обратно в 4-битные данные в реальном времени.
* **Результат:** Полученные данные о реальных оборотах (RPM) обрабатываются PID-регулятором в `dshot.c`. Контроллер динамически корректирует тягу, компенсируя просадки напряжения АКБ и сопротивление среды.

## 🛰 Система связи и шина "myCode"

Микроконтроллер работает в режиме **Slave** (ведомый), взаимодействуя с Raspberry Pi через USART1. Вся логика этого взаимодействия сосредоточена в директории `myCode`.

### Архитектура программной шины
* **Протокол:** Реализует кастомную программную шину на базе библиотеки `hydrolib` для структурированной передачи данных.
* **Надежность:** Весь трафик кодируется методом **COBS** (Consistent Overhead Byte Stuffing) и проверяется контрольной суммой **CRC8**, что исключает выполнение ошибочных команд.
* **Интерфейс памяти:**
    * **Команды:** Прием целевых значений тяги для 10 движителей и 4 вспомогательных PWM-выходов.
    * **Телеметрия:** Передача данных о напряжении двух линий АКБ и состоянии физического размыкателя (**Kill-switch**) на порту PA3.

## 🛠 Сборка и прошивка

### Компиляция
Проект использует **CMake** и **Ninja** для быстрой кросс-компиляции.

```bash
# Настройка директории сборки
cmake -S . -B build -G Ninja

# Компиляция проекта
cmake --build build
```

### Прошивка
С использованием `st-flash`:
```bash
st-flash write build/SAUVC_T1_T8.bin 0x8000000
```
